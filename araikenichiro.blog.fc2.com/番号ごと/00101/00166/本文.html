チームみらいの議席獲得について

昨日の参院選で、チームみらいが比例代表で1議席を獲得した。党首は「ゴールではなくスタート。ポリシーを持って活動していきたい」と語っている。私もエンジニアの一人として、今後の活動を応援している。本記事では、政府のAI活用の今後について、私見を述べる。

選挙権を持ってからそれなりの年月が経ったが、その経験から、過度な期待はせず、着実に進めてほしいと考えている。

スタートと言える難しさ

まず、厳しいことを言えば、1議席を獲得したことは「ゴールではなくスタート」というのはその通りだが、議員になれても実際にはスタートラインにすら立っていないというのが現実だ。この「スタートライン」についてはXでも何度か投稿している。例えば、下記のような投稿である。

これが、私がAGIやASIをゴールではなくスタートラインと位置付けている理由である。実際には、まだスタートラインにすら立てていないのが現状だ。AGIやASIが実用化されて初めて、問題に（一緒に）取り組むことが可能になる。

— 新井健一朗@want to (@arai_kenichiro) December 15, 2024

チームみらいは「確定申告の自動化」を掲げているが、本当のスタートは、アプリやサービスが完成した時点ではない。実際にリリースされ、ある程度の利用者が目的を達成して初めて「スタート」と言える。

エンジニア視点では「リリース＝ゴール」と考えがちだが、ユーザー視点では「目的が達成されて初めてゴール」だ。途中で止まったり、一部しか使えなければゴールではない。利用者が実際に使い、目的を達成し始めてからが本当のスタートだ。

さらに、日々の運用・メンテナンスや法改正への対応も必要だ。つまり、「ゴールは存在しない」とも言える。

政府のAI活用の現状について

チームみらいの議席獲得によって政府のAI推進が加速することを期待する声もあるが、それは現状の政府の動向に対する不満の裏返しでもある。

チームみらいの党首で今回議席を獲得した安野貴博党首は様々な経歴を持っているが、メディア出演時には松尾研に所属していたことがよく紹介される。その松尾研の松尾豊教授は内閣府「AI戦略会議」の座長として政府の中枢で行政に携わっている。

もちろん、座長と参議院議員、松尾教授と安野党首の違いや「テクノロジーで政治を変える」といった違いは多いが、政府はAI戦略を何もしてこなかったわけではない。松尾教授の言葉を借りれば、今の日本政府の現状は「満点」とのことだ。ただし、私の解釈では、これは不満がないという意味ではなく、過去と比べて、あるいは限られたリソースの中でという意味だと捉えている。

諸外国に比べれば「1/100の規模」と言っていることから、予算などのリソースが足りないということだろう。松尾教授の言葉からは、Xなどで流れる日本政府のAI政策への不満とは異なる印象を受ける。

また、動画で松尾教授と共に登壇している平将明議員は、デジタル大臣（第5・6代）、デジタル行財政改革担当大臣、行政改革担当大臣、サイバー安全保障担当大臣、内閣府特命担当大臣（規制改革）を兼任しており、政府中枢でAI政策のトップと言ってよい存在だ。平大臣とも関係がある状況が今の現状だ。

繰り返しになるが、安野党首との違いは多いものの、日本も決して素人だけが政策に携わっていたわけではない。したがって、チームみらいが永田町に行けば大きく改善するというのは期待しすぎだと考える。

その期待が大きいほど、成果とのギャップを感じて長続きしないというのは、過去の政治や行政でも一有権者として失敗例を見てきた。これは政府や行政だけでなく、一般的な商品や企業にも当てはまる。最近で言えば、電気自動車やソーラーパネルなどが失敗例だと感じている。

政府のOSSの活用について思うこと

非エンジニアからすればOSSは目新しいものに映るかもしれないが、エンジニアにとってはOSSは日常であり、昔から存在している。私は特にJavaプログラマなので、使用しているほとんどがOSSだ。

  • Java
  • Eclipse
  • Maven
  • Git
  • PostgreSQL
  • Apache
  • その他ライブラリやAPI

政府や行政は保守的で最新技術に疎いというイメージがあるかもしれないが、むしろ最新技術を早く導入しすぎて失敗している印象が強い。大手企業は逆に慎重だ。

20年以上前から自治体や教育現場でLinuxやOpenOffice.orgを導入する記事を見かけたが、結局はWindowsやMS-Officeに戻ることが多い。最近の生成AIで言えば、三豊市の「ごみ出し案内」などが例だ。

2023年12月の記事「ChatGPTでの業務効率化を“断念”──正答率94％でも「ごみ出し案内」をAIに託せなかったワケ　三豊市と松尾研の半年間」で、半年とあることからChatGPT 3.5登場から約半年後には試していたことになる。私からすれば、政府や行政は予算消化の特徴からむしろ早すぎると感じる。民間であればリターンが求められるため慎重だが、政府や行政はそれがないため、早く、早すぎて失敗している印象だ。

政府は昔から有識者を集めて理想を掲げてきたため、考え方が古いということはない。例えば、第五世代コンピュータ（今でいうAGIを創るような国家プロジェクト）、e-Japan戦略（IT、電子政府）、ユビキタス社会、最新の2050年ムーンショット目標も、一般人から見ればかなり高い目標だ。

デジタル庁や内閣府「AI戦略会議」もそうで、決して最近になってテクノロジーを重視し始めたわけではない。DXについても総務省が掲げる定義をまともに理解している民間のIT担当者がどれほどいるか。リモートワーク推奨やDX推進も、基本的には政府や行政が常に早い。民間が遅いのは、行政が予算ベースで動くのに対し、民間は売上や利益を確保する必要があるためで、民間が悪いわけではない。

※追記：初回投稿日同日19:40分。OSSについて補足すると、OSS自体はオープンソースソフトウェアのことであり、簡単に言うとサービス・アプリではない。それを利用するには別途構築することが前提となる。それをみんなが利用する場合は、クラウドか自前でサーバを構築する必要がある。YouTubeやGitHubのように利用者に使うには別途お金がかかると思った方がよい。自分のPCやスマホで動かす場合には問題ないため、お金のかかり方はどのような提供方法かにはよる。つまり、チームみらいがクラウドとして提供する場合は開発費以外にもお金がかかるのであり、これは運用している限り掛かることになる（毎年）。無料クラウド・サーバもあるが、基本的には個人向けであるため現実的ではない。何が言いたいかと言えば、開発・保守費だけでなく、運用も血税が必要ということである。これが、1政党内で負担するのか、新たに予算を設けるのかというのは気になるところである。

まとめ

  • チームみらいの考え方は素晴らしいが、過度な期待は禁物だ。
  • 有権者も現実的な視点を持ち、自分にできることを考えていくことが大切だ。
  • 技術的な課題よりも、運用や制度面の課題に注目し、着実な進歩を見守るべきだ。

「プログラマは仕様が決まりさえすれば実装できるの真意」でも書いたが、技術的な問題は生成AI以前からほとんど問題ではなく、それ以外の部分が課題である。その点がチームみらいではまだ想定されていないと感じるため、そこは暖かく見守る必要がある。

また、政治家は血税で動いている、国民のために活動しているという点を忘れず、当選後は大きすぎる目標ばかりでなく、堅実な目標を掲げてほしい。期待が大きすぎると失望も大きくなる。

確定申告の自動化が今年度にできるのか、来年度なのか。今年度からできる範囲はどこまでか。例えば、住宅ローン控除は会社員からの要望が多いが、全体から見れば少数なので後回しになるかもしれない。それを期待していた人は恩恵を受けられない。（年末調整がある）会社員は住宅ローン控除は1年目だけの申請なので、来年度対応ではメリットがない。
