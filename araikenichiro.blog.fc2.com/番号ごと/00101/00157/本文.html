リレーショナルデータベース入門の本について

前回から リレーショナルデータベース入門: データモデル・SQL・管理システム・NoSQL を読み進めている。前回は、「アカデミック的な構成になっているのでそれと比較すると読みやすい。本格的なアカデミック本はもっと難しいので中間的だろうけど。」と書いたが、実際は本格的なアカデミック本でした。著者の経歴もまさにアカデミックの経歴でした。

読む前にパラパラと捲ったときや最初読んでいるうちはちょっと読むのに時間がかかるか程度と思ったが、命題、定義、証明が増えていくにつれて今までのペースで読むの難しいと思い、現在はペースを大幅に下げている。もともと前回の本などは「 特大サイズ 書見台 高さ調節可能 wishacc直立竹ブックスタンド、クックブックホルダー 読書用（39.1 x 27.9cm） 」を机に置き、走りながら読んでいたが、それはさすがに厳しく今は走る用の本は別に変えて、この本はじっくりと読むことにした。管理も1日何ページとかではなく、1日15分に変更している。

集合の記号はたまに見るが、読み方をいつも忘れてしまうため、今回はそれをまとめた。本書の定義もあるが、まずは一般的な集合の記号をまとめている。

集合の記号の読み方

記号 読み方 意味（概要） 例・備考
∈ 要素である x ∈ A は「x は集合 A の要素」 社員123 ∈ 社員集合
∉ 要素でない x ∉ A は「x は A に含まれない」 社員999 ∉ 社員集合
⊆ 部分集合 A ⊆ B は「A は B に含まれる」 エンジニア ⊆ 社員
⊂ 真部分集合 A ⊂ B は「A ⊆ B かつ A ≠ B」 若手社員 ⊂ 社員
⊇ 包含する A ⊇ B は「A は B を含む（B ⊆ A）」 社員 ⊇ エンジニア
= 等しい A = B は「全く同じ集合」 男性社員集合 = 男性職員集合
∪ 和集合（または） A ∪ B は「A または B に含まれる」 営業 ∪ 技術 = 全社員
∩ 共通集合（かつ） A ∩ B は「A と B の両方に含まれる」 正社員 ∩ 技術者 = 技術系正社員
−（または \） 差集合 A − B は「A にあって B にないもの」 社員 − 管理職 = 非管理職
× デカルト積 A × B は「A と B の全組み合わせ」 社員 × プロジェクト（CROSS JOIN）
∃ 存在する 「ある x が存在する」 ∃x ∈ 社員: x.役職 = '課長'
∀ すべて 「全ての x に対して」 ∀x ∈ 顧客: x.年齢 ≥ 20
⇒ ならば（含意） P ⇒ Q は「P が真なら Q も真」 x ∈ 技術者 ⇒ x ∈ 社員
⇔ 同値（ならばかつその逆も） P ⇔ Q は「P ならば Q、かつ Q ならば P」 x ∈ 管理職 ⇔ x.役職 = 'マネージャ'
¬ 否定（でない） ¬P は「P ではない」 ¬(x ∈ 正社員) → 正社員ではない
∅ 空集合 要素が一つもない集合 退職者集合 = ∅
∨ 論理和（または） P ∨ Q は「P または Q が真」 x = '課長' ∨ x = '部長'
∧ 論理積（かつ） P ∧ Q は「P かつ Q が真」 年齢 ≥ 30 ∧ 役職 = '主任'
φ / ϕ ファイ ギリシャ文字。関数や変数名として使用 数学で論理式 φ(x) など
