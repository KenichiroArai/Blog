開発記録

GitHub Projectsに、kmgシリーズの開発を管理するプロジェクトの用意した。Projectsは「kmg」で公開している。下記の画像のような状態だ。

Statusは、アジャイル開発、特にスクラムを参考にして作成した。あくまでもアジャイル開発風として一部の参考にとどめる。

  1. new issues
    誰かが思いついた、または外部／チームから上がってきた未整理の課題を一時的に置く場所。
    → ここで担当者／優先度／分類／必要なら軽いスコーピングを付与し、次のステップへ。
  2. icebox
    アイデア段階・将来的にやるか検討中のもの。すぐ着手しないが捨てない候補。
    定期的（例：週1）に見直し、価値が上がったら backlog へ昇格。
  3. backlog
    着手予定が確定しているが、まだ手をつけていない課題。
    優先順位付け・見積もり（ポイント / 時間）を付け、スプリントや次の着手候補としてキープ。
  4. in progress
    誰かが実際に手を動かして作業中。
    ブランチを切って開発し、進捗を都度コメント。途中での大きな軌道修正があればここで明記。
  5. review/qa
    実装が完了し、プルリクエストが出ている状態。
    コードレビュー・自動テスト・ステージ環境でのQAを経て承認を得る。
  6. done
    マージ済み／本番反映済み／完了を確認した課題。
    （必要ならクローズとアーカイブのルールを明記）
